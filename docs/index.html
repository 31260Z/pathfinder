Pathfinder

Movement library for VEX V5 robots running PROS — pose tracking, calibration, controllers, and autonomous primitives. Same family as LemLib / EZ-Template, more ambitious in scope.

C++20 PROS 4.2 VEX V5 Header-only 45 tests passing

Pose Tracking

6-DOF state estimator (pose + body velocity). Three tiers: dead reckoning, EKF, and MCL against a field map.

Autonomous Primitives

moveTo, moveToPose (Boomerang), turnTo, swingTo, and spline-based follow with waypoint annotations.

Drift-Aware Control

Cross-track regulator uses body-frame lateral velocity for lookahead prediction. Motion chaining inherits drift between segments.

Automatic Calibration

WheelFinder finds tracking-wheel offsets. AutoPid synthesizes PID gains from step-response data. DriftCoeff measures lateral friction.

Declarative Sensors

Builder API for tracking wheels, IMUs, distance sensors, and landmarks. Full PROS adapter suite.

Host-Testable Math

Header-only, host-buildable. All controllers, odometry, and geometry are tested off-robot with doctest (45/45 passing).

Quick Start

#include "pathfinder/pathfinder.hpp"

// 1. Build your drivetrain config
auto drive = pathfinder::Drive{
    .left_motors  = pathfinder::MotorGroup::from_pros({-1, -2, -3}),
    .right_motors = pathfinder::MotorGroup::from_pros({4, 5, 6}),
    .track_width_in = 12.5,
    .wheel_diameter = 2.75,
    .gear_ratio = pathfinder::GearRatio::green,
    .horizontal_drift_factor = 6.0,
};

// 2. Declare sensors (wheels + IMU built internally)
auto sensors = pathfinder::SensorBuilder{}
    .with_left_motors({-1, -2, -3})
    .with_right_motors({4, 5, 6})
    .with_imu(7)
    .build();

// 3. Create the chassis
auto chassis = pathfinder::Chassis(drive, sensors);
chassis.resetPose({0, 0, 90});   // start center, facing right

// 4. Autonomous
void autonomous() {
  chassis.moveTo({24, 0});
  chassis.turnTo(180);
  chassis.follow({{{24, 0}}, {{48, 24}}, {{72, 0}}});
}

// 5. Driver control
void opcontrol() {
  double left  = controller_get_analog(ANALOG_LEFT_Y);
  double right = controller_get_analog(ANALOG_RIGHT_X);
  chassis.opcontrol(pathfinder::TankDrive{left, right});
}

Geometry

2D math primitives — all hand-rolled, no external dependencies.

Vector2

2D vector in inches. Full arithmetic, dot, cross, rotate, normalize, distance.

pathfinder::Vector2 v = {12.0, 5.0};
double mag = norm(v);           // 13.0
auto u = normalize(v);          // unit vector
auto r = rotate(v, M_PI_2);    // 90 CCW

Pose2

Pose (x, y inches; heading as Angle). Compose, inverse, transform between frames.

pathfinder::Pose2 p = {24.0, 0.0, pathfinder::Angle::degrees(90)};
auto world = p.transform({5.0, 0.0});     // local bot offset to world
auto local = p.inverse_transform(world);  // world to bot frame

Angle

Wrapper around radians with degree accessors and normalization.

auto a = pathfinder::Angle::degrees(370);
a.normalize_unsigned();    // 10
a.normalize_signed();      // 10
auto delta = shortest_angle(from, to);  // signed diff in (-pi, pi]

Bot

Describes the robot’s geometry: footprint, origin corner, and named points.

auto bot = pathfinder::Bot{}
    .footprint(15.0, 14.5)
    .origin(pathfinder::Corner::BackLeft)
    .point("intake", forward=5, right=10)
    .point("elevator", forward=0, right=0);

NamedPoint parameter DSL

The named-parameter syntax forward=, backward=, right=, left= resolves to a 2D offset at compile time, with mutual-exclusivity enforcement.

Autonomous

Chassis::moveTo — drive to a point

chassis.moveTo({24, 0});                                         // drive 24" forward
chassis.moveTo({0, 24}, pathfinder::MoveOpts{ .reverse = true }); // drive backward

Uses MoveToPoint internally: along-track PID + cross-track PID + heading PID. Drift-aware: cross-track regulator factors in lateral body velocity.

Chassis::moveToPose — drive to a pose (Boomerang)

chassis.moveToPose({48, 12, pathfinder::Angle::degrees(90)});
chassis.moveToPose(target, pathfinder::MoveOpts{ .lead = 0.5, .reverse = false });

Boomerang controller: drives toward a “carrot” projected beyond the target, then turns to the target heading. lead controls how far the carrot projects (0.3 default; higher = earlier turn).

Chassis::follow — spline path with pure pursuit

// Simple Vector2 waypoints
chassis.follow({{{0, 0}}, {{24, 0}}, {{48, 24}}});

// Rich waypoints with metadata
chassis.follow({
    pathfinder::Waypoint{.position_x = 0,  .position_y = 0,   .speed_cap_ips = 30},
    pathfinder::Waypoint{.position_x = 24, .position_y = 0,   .speed_cap_ips = 20},
    pathfinder::Waypoint{.position_x = 48, .position_y = 24,  .heading_deg = 90},
});

Waypoints define a centripetal Catmull-Rom spline, tracked via pure pursuit. Each waypoint supports:

FieldDescription
speed_cap_ipsOptional speed limit at this waypoint
heading_degSoft heading hint at this waypoint
on_arriveCallback fired when the robot reaches this waypoint
must_pass_throughIf false, the waypoint only influences the spline shape (default true)

Chassis::turnTo

chassis.turnTo(90);                                   // absolute heading
chassis.turnTo({48, 0});                              // face a point
chassis.turnTo(90, pathfinder::TurnOpts{ .direction = pathfinder::Direction::CW });

Chassis::swingTo

chassis.swingTo(90, pathfinder::Side::Left);          // pivot on left side
chassis.swingTo({48, 0}, pathfinder::Side::Right);    // face point, pivot on right

Odometry

Three-tier localization, all behind the polymorphic ILocalizer interface.

TierClassWhen to use
1DeadReckoningDefault — good enough for 99% of matches. Wheel + IMU integration.
2EkfWhen you have GPS or landmark inputs. Extended Kalman Filter on full 6-DOF state.
3MclGlobal localization with a field map. Monte Carlo (particle filter).

DeadReckoning

pathfinder::DeadReckoning::Config config{
    .parallel_wheel_y_offset_in = 5.5,
    .perp_wheel_x_offset_in = -3.0,
    .has_perp_wheel = true,
    .lateral_friction_coefficient = 5.0,
};
pathfinder::DeadReckoning dr(config, initial_pose);
dr.update(par_delta_in, perp_delta_in, heading_rad, dt_sec);
auto pose = dr.pose();
auto vel  = dr.body_velocity();  // {v_x, v_y, omega} in body frame

pathfinder::Ekf::Config config{

Calibration

WheelFinder

Find tracking-wheel offsets by pinning bot corners and rotating in place. Runs 4 pivots (one per corner) and aggregates estimates.

auto finder = pathfinder::calibration::WheelFinder(chassis);
auto result = finder.run_all_corners();     // blocks, no motor control needed
if (result.converged) {
    // result has per-wheel mean + stddev offsets
}

AutoPid

Open-loop step-response system ID → first-order-plus-dead-time (FOPDT) model fit → IMC PID gain synthesis → closed-loop verification.

auto tuner = pathfinder::calibration::AutoPid(chassis);
auto [lateral, angular] = tuner.tuneBoth();
if (lateral.converged) {
    // lateral.gains.kP, .kI, .kD are ready
    // lateral.model.K, .tau, .L hold the FOPDT fit
}

DriftCoeff

Auto-measure lateral friction coefficient μ. Accelerates forward, rotates to induce lateral drift, coasts, and fits an exponential decay to the lateral velocity trace.

auto coeff = pathfinder::calibration::DriftCoeff(chassis);
auto r = coeff.measure();     // r.mu_per_sec, r.converged

Chassis

The Chassis class is the central integration point. It owns the drivetrain, sensors, localization estimator, controller stacks, and telemetry sink.

Construction

auto chassis = pathfinder::Chassis(
    left_motor_group,
    right_motor_group,
    bot_def,
    sensors,
    pathfinder::Localization::DeadReckoning,  // or Ekf / Mcl{...}
    drive_config
);

Pose & calibration

chassis.calibrate();                           // wait for IMUs (5s timeout)
chassis.setPose(0, 0, 90);                     // teleport pose
auto pose = chassis.getPose();                 // Pose2
auto cov  = chassis.getPoseCovariance();       // Matrix3 (zero for DR)
auto vel  = chassis.getBodyVelocity();         // BodyVelocity

Async stubs (v0.1 throws)

chassis.waitUntil(6.0);   // throws runtime_error
chassis.waitUntilDone();   // throws runtime_error
chassis.cancelMotion();    // best-effort zero motors (safe)

Direct motor control

chassis.applyVoltage(6000, 6000);  // raw per-side mV (for calibration utils)

Inspection

chassis.bot().point("intake");       // NamedPoint
chassis.drive().track_width_in;      // Drive fields
chassis.sensors().tracking_wheels(); // sensor list
chassis.active_verb();               // current motion verb name
chassis.last_left_voltage_mv();      // last commanded mV

Testing hook

auto result = chassis.tick(dt_sec);  // full sensor-odom-motor iteration
// result.pose, result.cmd, result.done

Controller Stack

Each axis (lateral, angular) has a full controller stack: feedback PID + feedforward + optional motion profile + slew limiter.

using namespace pathfinder;

chassis.setController(ControllerAxis::Lateral, ControllerStack{
    .feedback = Pid::Gains{ .kP = 1.2, .kD = 0.15 },
    .ff       = FeedForward{ .kV = 0.18, .kA = 0.03, .kS = 0.0 },
    .profile  = motion_profile::Trapezoidal{ .v_max = 36.0, .a_max = 24.0 },
    .slew     = SlewSpec{ .max_rate_per_sec = 200.0 },
});

chassis.setController(ControllerAxis::Angular, ControllerStack{
    .feedback = Pid::Gains{ .kP = 2.5, .kD = 0.3 },
    .ff       = FeedForward{ .kV = 0.10, .kA = 0.01, .kS = 0.0 },
    .slew     = SlewSpec{ .max_rate_per_sec = 400.0 },
});
FieldDefaultDescription
feedbackPid::Gains{1.0, 0.0, 0.1}PID feedback gains
ffFeedForward{0.15, 0.02, 0.0}Feedforward (kV, kA, kS)
profileoptionalTrapezoidal motion profile (position control)
s_profileoptionalS-curve motion profile (smooth, jerk-limited)
slewSlewSpec{}Rate-of-change limiter (0 = no limit)

Localization Tiers

Choose localization via a tag type passed to the Chassis constructor.

// Tier 1: Dead reckoning (default — no config needed)
pathfinder::Chassis chassis(l, r, bot, sens, pathfinder::Localization::DeadReckoning, drive);

// Tier 2: Extended Kalman Filter
pathfinder::Chassis chassis(l, r, bot, sens, pathfinder::Localization::Ekf, drive);

// Tier 3: Monte Carlo (particle filter)
auto field = pathfinder::FieldMap::default_perimeter();
pathfinder::Chassis chassis(l, r, bot, sens,
    pathfinder::Localization::Mcl{ .field_map = field, .particles = 300 },
    drive);
